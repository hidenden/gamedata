# Annotationレベル基準

## 論理的な分類

- 0-9: 数年に一度のイベント(調整: 累計グラフに大きな影響を及ぼした出来事)
- 10-19: 年一のイベント (調整: 累計グラフに中程度､週販グラフに大きな影響を及ぼした出来事)
- 20-29: 四半期一イベント (調整: 週販グラフに中程度の影響を及ぼした出来事､累計グラフへの影響はほぼ無い)
- 30-39: 月一イベント (調整: 週販グラフに影響はないが著名な出来事)
- 40-50以上: 週イベント (調整: 週販グラフに影響がないマイナーな出来事)

## 表示視点での分類

- 0-9: 10年以上の長期のグラフでも表示する出来事
- 10-19: 2年~9年(Switch, PS5,4の累計など)で表示する
- 20-29: 1年~2年(100週間)程度のグラフで表示する
- 30-39: 半年~1年程度の週販グラフで表示する
- 40-50以上: 週販の拡大グラフで表示する

## 分類例

### 0-9: 数年に一度のイベント

- 例: 新ハードウェアの発売
- 例: 任天堂のハードウェアの価格改定(任天堂はレアなので)
- 例: 業界の数年に一度の変化､イベントなど
- 例: 100万本以上級のゲーム発売日

### 10-19: 年一のイベント

- 例: (任天堂以外の)ハードウェア価格改定
- 例: ハードウェア新モデル発売
- 例: 重要な周辺機器発売
- 例: 50-100万本級のゲーム発売日
- 例: 数100万本級ソフトの大型のアップデートや新機能のリリース
- 例: Nintendo Direct, Pokemon Presents (Nintendo Directは年に数回あるが､注目度が圧倒的に高いので年一イベントに分類)
- 例: PlayStation State of Play、MicrosoftのXbox Showcaseなどの定期的なイベント
- 例: E3、Gamescomなどの年次イベント(HW毎に話題が異なるので注意)

### 20-29: 四半期一イベント

- 例: 20-50万本級のゲーム発売日
- 例: 100万本級のゲームのアップデートや新機能のリリース
- 例: 大型セールイベント（例: ブラックフライデー、サマーセールなど）
- 例: マイナーな周辺機器発売､刷新


### 30-39: 月一イベント

- 例: 10-20万本級のゲーム発売日
- 例: 50万本級のゲームのアップデートや新機能のリリース
- 例: 定期的なアップデートや新機能のリリース、月次のセールイベントなど
- 例: 年次のセールイベント、定期的なアップデートや新機能のリリースなど

### 40-50以上: 週イベント

- 例: 10万本未満のソフトの発売日
- 例: 業界の話題(ただしゲーム機に紐づくものに限る)
